## 一、项目概述

### 1.1 项目目标

企业内部日常工作管理工具，替代 Excel 记录需求和任务，支持多用户协作和权限管理。

### 1.2 MVP 范围（第一阶段）

**本期实现**：登录 + 日常任务登记模块

- 用户注册登录（防止未登录直接访问）
- 日常任务登记（CRUD）
- 任务已完成/未完成切换
- 任务顺延到下一天（未完成的自动复制到明天）

**预留功能**（下期实现）：
- 需求登记模块
- 权限分配（管理员给用户设置权限）
- 用户后台新增
- Excel 导入

### 1.3 用户角色

| 角色 | 描述 |
|------|------|
| 管理员 | 创建和管理用户、分配权限 |
| 普通用户 | 使用任务登记功能 |

## 三、数据模型

| 字段 | 类型 | 说明 |

| id | INT | 主键，自增 |

| updated_at | DATETIME | 更新时间 |

### 3.3 任务明细表（TaskDetail）

| 字段 | 类型 | 说明 |

| remark | TEXT | 备注，可空 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 四、功能详情

### 4.1 认证模块

| 功能 | 描述 |
|------|------|
| 用户注册 | 用户名 + 密码（仅管理员可操作，预留） |
| 用户登录 | 用户名 + 密码，返回 JWT Token |
| 登录保护 | 所有 API 需要 Bearer Token |
| Token 刷新 | 过期前自动刷新 |

### 4.2 任务模块

| 功能 | 描述 |
|------|------|
| 任务列表 | 显示当前用户的任务，分页 |
| 切换视图 | 已完成 / 未完成 两个 Tab |
| 新增任务 | 填写任务名称、登记时间（默认当前） |
| 修改任务 | 所有字段可修改 |
| 标记完成 | 点击标记为已完成，自动记录完成时间 |
| 删除任务 | 物理删除 |
| 查询搜索 | 按任务名称、执行人、登记时间查询 |
| 排序 | 按登记时间、完成时间排序 |
| 任务明细 | 每个任务可展开，添加明细（进度、时间、备注） |
| 顺延到明天 | 未完成的任务，一键复制到明天，保留原记录 |

### 4.3 任务顺延逻辑

当用户点击"顺延到明天"：
1. 原任务标记为已完成（complete_time = 当天）
2. 创建新任务：
   - name = 原任务名称
   - register_time = 明天 00:00
   - publisher = 原任务 publisher
   - is_completed = false
3. 任务明细一起复制到新任务

## 五、API 设计

### 5.1 认证接口

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/auth/register | 注册用户（预留） |
| POST | /api/auth/login | 登录 |
| POST | /api/auth/refresh | 刷新 Token |

### 5.2 任务接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/tasks | 获取任务列表（支持分页、筛选） |
| GET | /api/tasks/{id} | 获取单个任务 |
| POST | /api/tasks | 创建任务 |
| PUT | /api/tasks/{id} | 更新任务 |
| DELETE | /api/tasks/{id} | 删除任务 |
| POST | /api/tasks/{id}/complete | 标记完成 |
| POST | /api/tasks/{id}/defer | 顺延到明天 |

### 5.3 任务明细接口

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/tasks/{task_id}/details | 获取任务明细 |
| POST | /api/tasks/{task_id}/details | 添加任务明细 |
| PUT | /api/tasks/{task_id}/details/{id} | 更新任务明细 |
| DELETE | /api/tasks/{task_id}/details/{id} | 删除任务明细 |
